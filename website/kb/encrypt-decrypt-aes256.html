How to Encrypt/Decrypt Files with AES-256

Internal-style KB entry based on OpenSSL lab work and a Python/Tkinter encryption utility — written the way a help desk knowledge base should read.

Encrypt

openssl enc -aes-256-cbc -salt -pbkdf2 -in "file.zip" -out "file.zip.enc" -k "YourPassphrase"

Decrypt

openssl enc -d -aes-256-cbc -pbkdf2 -in "file.zip.enc" -out "file.zip" -k "YourPassphrase"

The GUI wrapper

Typing OpenSSL flags correctly every time is easy to get wrong, so this is wrapped in a small Python/Tkinter desktop tool with Browse, Encrypt, and Decrypt buttons — the same two commands above, run from a form instead of a terminal:

def encrypt_file(file_path, password):
    encrypted_file = file_path + ".enc"
    command = f'openssl enc -aes-256-cbc -salt -in "{file_path}" -out "{encrypted_file}" -k "{password}"'
    result = subprocess.run(command, shell=True, capture_output=True, text=True)
    if result.returncode == 0:
        messagebox.showinfo("Success", f"File encrypted: {encrypted_file}")
    else:
        messagebox.showerror("Error", f"Encryption failed: {result.stderr}")

The GUI's own commands predate the -pbkdf2 flag shown above — a good reminder that a tool is only as current as the code inside it. A file encrypted with one version and decrypted with mismatched flags on the other end produces exactly the bad decrypt error covered below.

Common issues

SymptomCauseFix
bad decryptWrong password or -pbkdf2 mismatchMatch flags on both commands; verify passphrase
Corrupt outputFile transferred in text modeTransfer .enc as binary attachment
Unknown optionOlder OpenSSLUpgrade OpenSSL or drop -pbkdf2 on both sides

Security notes

  • Never send passphrase in the same channel as the encrypted file.
  • For routine use, prefer BitLocker or company-approved secure transfer over ad hoc OpenSSL.
