# Add a project to your portfolio

Use this when you finish portfolio project #2 (or client work you can show).

---

## 1. Website (`website/index.html`)

Copy the first `<article class="project-item">` block in the **Selected work** section. Paste it below SecPlus Sensei and update:

| Field | Example |
|-------|---------|
| Thumbnail | `assets/my-project-icon.png` (add image to `website/assets/`) |
| Meta line | `Web app · 2026` |
| Title | Your project name |
| Description | 2–3 sentences — what it does, what you built |
| Stats (optional) | 3 highlight numbers |
| Tags | 3–5 tech tags |
| Links | GitHub, live demo, case study |

Remove the "Coming soon" block if you added one back.

Push to `main` — GitHub Pages redeploys automatically.

---

## 2. GitHub README (`README.md`)

Add a bullet under **For recruiters** or a second project subsection linking to the new repo.

Optional: add screenshots to `docs/screenshots/` and update the Screenshots section.

---

## 3. Freelance profiles

Add the new project to:
- LinkedIn → Projects
- Upwork → Portfolio
- Fiverr → Gallery / gig examples

Use the same 2–3 sentence description + link. Point to your portfolio site as the hub:

`https://jacobbarrera2024-sketch.github.io/secplus-sensei/`

---

## 4. Case study (optional, high impact)

Copy `docs/PORTFOLIO.md` structure for the new project:
- Problem
- What you built
- Technical highlights
- By the numbers
- What it demonstrates

Link from the website project card.

---

## Thumbnail tips

- **App icon** — 512×512 PNG, drop in `website/assets/`
- **Screenshot** — crop to square or use as wide image in a future layout upgrade
- Keep visuals **neutral** — they don't need to match other projects' UI themes
