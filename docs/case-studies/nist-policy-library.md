# NIST Security Policy Library — Case Study

> **Type:** Academic / self-directed lab project (Full Sail, B.S. Cybersecurity — *Project and Portfolio IV*). Not paid employment.

## Problem

Small organizations frequently have no documented, standards-aligned security policies — which means no consistent way to enforce password strength, approve infrastructure changes, or onboard new employees securely. Auditors and help desk teams both need these documents to exist before they can be followed.

## Steps

1. Researched applicable **NIST guidance** (800-53 access control family, 800-63 digital identity guidelines) for password and account management requirements.
2. Authored a **Password Standard** — complexity, length, rotation, and reuse rules aligned to current NIST guidance (moving away from outdated "change every 90 days" thinking where appropriate).
3. Authored a **Change Control Management Policy** — a defined request → review → approval → implementation → documentation workflow for infrastructure changes.
4. Authored a **New Account Creation Procedure** — a step-by-step process for provisioning new-hire accounts with least-privilege access and an audit trail.

## Fix

Turned each policy into an actionable procedure a technician could actually follow — not just a compliance document. Each includes explicit ownership (who approves, who executes) and enforcement points.

## Result

A ready-to-use, three-document policy library. The New Account Creation Procedure and Change Control Policy map almost directly onto the tickets a help desk technician processes every day: provisioning access, and getting sign-off before making a change.

## Tools

NIST 800-53 / NIST 800-63 frameworks · Microsoft Word · policy & procedure writing
